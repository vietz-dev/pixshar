# Pixshar — Claude Code Session Prompt

## What we're building
Pixshar is a self-hostable, private event photo sharing app. The photographer (admin) creates
password-protected galleries, uploads photos, and shares a link. Guests unlock the gallery with
a password, browse photos, and can upload their own.

## Stack
- **Frontend**: Next.js 15 (App Router, `output: standalone`)
- **Component Library**: Shadcn/ui (Tailwind-based, no pre-built components used except icons)
- **Backend**: Hono + Bun + Effect
- **Auth**: BetterAuth — single admin account (seeded, no public sign-up), per-gallery JWT cookies for guests
- **ORM**: Prisma + SQLite
- **Storage**: S3-compatible via AWS SDK (Minio bundled for self-hosting)
- **Infra**: Docker Compose + Helm chart

## Key architecture decisions (don't revisit these)
- **Presigned URLs**: photos are never proxied through the API; the browser fetches directly
  from S3/Minio using time-limited presigned URLs generated by the API
- **Gallery sessions**: guests get a JWT cookie scoped to a single event after entering the
  password — no user accounts
- **Image processing**: eager, in-process (Effect.fork), not queued. On upload: store original
  → resize to 1920px display + 400px thumb in parallel → upload both to S3 → mark PROCESSED.
  Retry with exponential backoff via Effect.
- **One S3 bucket, path-prefixed**: `{eventId}/originals/`, `{eventId}/display/`, `{eventId}/thumbs/`
- **Single admin**: BetterAuth with `disableSignUp: true`, admin created via seed script on
  first container start

## Data model (Prisma)
```
User         — BetterAuth managed (+ Session, Account, Verification tables)
Event        — id, slug, name, description, passwordHash, status (PROCESSING|READY), createdById
Photo        — id, eventId, photographerName, originalKey, displayKey, thumbKey,
               status (PENDING|PROCESSED|FAILED), uploadedBy (ADMIN|GUEST)
```

## API routes
```
POST /api/auth/*                           BetterAuth (admin login/session)
GET  /api/events                           List events [admin]
POST /api/events                           Create event [admin]
GET  /api/events/:id                       Get event + photos [admin]
DELETE /api/events/:id                     Delete event + S3 cleanup [admin]
POST /api/gallery/:slug/unlock             Password check → set gallery cookie
GET  /api/gallery/:slug                    Gallery + presigned URLs [gallery session]
POST /api/gallery/:slug/upload             Guest upload [gallery session]
POST /api/upload/events/:id/photos         Admin upload [admin]
GET  /api/upload/events/:id/photos/status  Processing status poll [admin]
GET  /health                               Health check
```

## What's done
- Full API implementation (all routes above)
- Effect-based image processing pipeline with retry
- BetterAuth config (admin-only, gallery JWT middleware)
- Prisma schema + seed script
- Docker Compose (api + web + minio + minio-init)
- Helm chart (values-driven, BYO S3 support)
- Next.js pages: login, admin dashboard (event list), gallery password gate
- Shared types package

## Coding conventions
- TypeScript strict mode throughout — no `any`
- Effect for any async pipeline that can fail and should retry
- Zod for all external input validation (API request bodies)
- No UI component library — plain CSS with CSS variables (defined in `globals.css`)
- CSS variables: `--bg`, `--surface`, `--border`, `--text`, `--text-muted`, `--accent`, `--danger`, `--radius`
- Next.js: `"use client"` only where interactivity is needed; prefer server components for
  data fetching where it makes sense
- API responses: always `{ error: string }` on failure with appropriate HTTP status
